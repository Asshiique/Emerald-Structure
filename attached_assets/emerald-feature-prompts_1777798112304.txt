PROMPT 4 — PUSH NOTIFICATIONS
(Paste this last — most complex)
----------------------------------------------------------------

Add push notification support to the Emerald International School app.

Use Expo Notifications (expo-notifications library) for this feature.

PART A — Setup:
- Install and configure expo-notifications
- On app first launch, ask user permission for notifications
- Save the device push token to the user's database record (field: pushToken)
- Handle notifications when app is in foreground, background, and closed state

PART B — Notification triggers:
1. When admin posts a new notice → send push notification to all users with title = notice title and body = first 100 characters of notice body
2. When a new homework is posted → send to students in that class section with title "New Homework" and body = subject + assignment title
3. When fee is overdue (status = overdue) → send reminder notification with title "Fee Reminder" body "Your Q[X] fee is overdue. Please contact school office."

PART C — Notifications settings screen:
- Add "Notifications" menu item in Profile page (already exists as placeholder, now make it functional)
- NotificationsSettingsPage:
  - Toggle switches for each notification type:
    * New notices (on by default)
    * New homework (on by default)
    * Fee reminders (on by default)
    * Event reminders (on by default)
  - Each toggle: label on left, Switch widget on right, red color when on
  - Save preferences to user's database record
  - "Test Notification" button at bottom — sends a test notification immediately to verify it's working

PART D — Notification badge:
- Show unread count badge on the Notices bottom nav icon
- Badge: small red circle with white number, positioned top right of the icon
- Count = number of notices where current user UID is NOT in the isRead array
- Update (disappear) when user visits NoticesPage

Use the existing app theme: primary red #C0282A, font Plus Jakarta Sans, border radius 14px for cards.
Keep all existing screens and features intact — only add new functionality.
