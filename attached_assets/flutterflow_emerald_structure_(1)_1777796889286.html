Firebase Auth setup
Enable these providers
Email/Password for admin and teachers. Phone (OTP) for parents — easier for Kerala parents who may not have Google accounts.
Auth console
Security rules (important)
Set Firestore rules so parents can only read their own child's data. Admins get full read/write. Never leave rules open.

Exact build sequence
1
Create FlutterFlow project
flutterflow.io → New Project → Blank. Choose mobile. Connect Firebase (it guides you step by step).
Day 1 · 1 hr
2
Set theme + colors + font
Theme Settings → add all 7 colors above. Add Plus Jakarta Sans. This makes every screen consistent automatically.
Day 1 · 30 min
3
Create Firestore collections
In FlutterFlow → Firestore → Create collections: users, notices, attendance, marks, fees, timetable. Add all fields.
Day 2 · 2 hrs
4
Build SplashScreen + LoginPage
Logo, school name, phone OTP login. Test sign in works before touching other pages.
Week 1
5
Build NavBar + empty pages
Add Bottom Nav Bar widget. Connect to 5 pages: Home, Notices, Progress, Fees, Profile. Just titles for now.
Week 1
6
Build reusable components
Create all 6 components listed in Widgets tab BEFORE building pages. Pages will use them as building blocks.
Week 2
7
Build NoticesPage first
Easiest data page. ListView + Firestore query + NoticeCard component. Once this works, other pages follow same pattern.
Week 2
8
Build HomePage
Most complex. Build stats, schedule strip, notice banner one block at a time. Test each block separately.
Week 3
9
Build Progress + Fees + Profile
These follow the same ListView + Firestore pattern. Should be faster now that you know the flow.
Week 3–4
10
Add Admin pages + test with real data
PostNoticePage is just a form. Add 5 test students to Firestore, test every screen with real data.
Week 4–5
11
Test on real Android phone
FlutterFlow → Run → Download APK. Install on Android. Give to 2–3 parents for feedback before Play Store.
Week 5–6
12
Publish to Play Store
FlutterFlow → Publish → Android. Needs Google Play Developer account ($25 one-time). Follow FlutterFlow's publish guide exactly.
Week 6–8
One rule to follow
Never skip to the next step if the current one isn't working. Especially login — fix it completely before building any data screen.
