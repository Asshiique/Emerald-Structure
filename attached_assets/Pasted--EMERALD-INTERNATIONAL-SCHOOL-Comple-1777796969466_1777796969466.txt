PROMPT 2 — FIREBASE COLLECTIONS SETUP
(Use in: FlutterFlow → Firestore → Create Collection → AI Gen)
----------------------------------------------------------------

Create the following Firestore collections for a school management app:

COLLECTION 1: users
Fields:
* uid (String) — Firebase Auth UID
* name (String) — full name
* role (String) — values: parent, student, teacher, admin
* classSection (String) — example: X-B
* rollNo (String) — example: EIS/2024/1024
* parentName (String)
* phone (String)
* email (String)
* profilePhoto (String) — image URL
* createdAt (Timestamp)

COLLECTION 2: notices
Fields:
* title (String)
* body (String)
* category (String) — values: Academic, Events, Fees, Sports, General
* isRead (List of Strings) — list of user UIDs who have read it
* postedBy (String) — admin UID
* targetRole (String) — values: all, parent, student, teacher
* createdAt (Timestamp)

COLLECTION 3: marks
Fields:
* studentId (String) — UID of student
* subject (String)
* score (Integer)
* totalMarks (Integer)
* examType (String) — values: Unit Test, Term Exam, Final
* term (String) — values: Term 1, Term 2, Term 3
* date (Timestamp)

COLLECTION 4: attendance
Fields:
* studentId (String)
* date (Timestamp)
* status (String) — values: present, absent, late
* markedBy (String) — teacher UID
* classSection (String)

COLLECTION 5: fees
Fields:
* studentId (String)
* quarter (String) — values: Q1, Q2, Q3, Annual
* amount (Integer)
* status (String) — values: paid, pending, overdue
* paidOn (Timestamp)
* receiptNo (String)
* dueDate (Timestamp)

COLLECTION 6: timetable
Fields:
* classSection (String)
* day (String) — values: Monday, Tuesday, Wednesday, Thursday, Friday
* slots (List) — each slot has: time (String), subject (String), teacherName (String)

----------------------------------------------------------------
GENERAL TIPS FOR USING THESE PROMPTS IN FLUTTERFLOW
----------------------------------------------------------------

1. Always set up Theme (Prompt 1) and Firebase Collections (Prompt 2) BEFORE generating any pages.

2. When using AI Gen in FlutterFlow:
   - Open the page
   - Click the wand/AI icon
   - Paste the prompt
   - Click Generate
   - Review and adjust — AI won't be 100% perfect, fix small things manually

3. After generating each page, immediately preview it on mobile to check layout.

4. Build and test in this order:
   Splash → Login → HomePage → NoticesPage → NoticeDetail → ProgressPage → FeesPage → ProfilePage → PostNoticePage

5. For Firestore queries in each page:
   - Go to the ListView widget
   - Set Backend Query → Firestore Collection
   - Choose the correct collection
   - Add filter: field "studentId" equals AppState.currentUserUID

6. To restrict admin pages:
   - Add a Conditional widget at the top of admin pages
   - Condition: currentUserDocument.role == "admin"
   - If false: show "Access denied" text or redirect to HomePage

7. Test with real data before publishing:
   - Manually add 3 test student documents in Firebase Console
   - Add 5 sample notices
   - Log in as different roles and verify each screen shows correct data
